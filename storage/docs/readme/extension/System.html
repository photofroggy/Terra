Extension: System

Included in: All packages
Default status: On

The System extension should be considered to be a core part of the bot. The extension provides a set of commands which are needed to take full control of some of the core features of your bot. There are also commands to show information about different things, including the environment that the bot is running in. Many of the commands included in this extension are Owner only, as you can do quite a lot with them.

This extension is contained in the "System.py" file, located in the folder ~/extensions, meaning it is loaded into the bot on startup.

!about

The about command! This simply shows very basic information about the bot.

!build

This command shows some brief information about the build the bot is running on.

!system

The system command shows quite a bit more information than the previous two commands. This command will show you information about the bot, the build it is using, and the environment it is running in.

!uptime

Use this command to see how long the bot has been running for.

!extensions

This command shows you a list of the extensions currently loaded into the bot.

!extensions reload

This command is for the Owner only.
Use this command to reload all of the extensions currently loaded in the bot.

!extension [name]

You can use this command to view information about an extension, where [name] is the name of the extension you wish to view information for. For example, typing !extension dAmn Manager shows information about the extension "dAmn Manager".

!ctrig [trigger]

This command is for the Owner only.
Use this command to change your bot's trigger character, where [trigger] is the desired trigger character. For example, to change the bot's trigger character to ":", type !ctrig :.

!autojoin

This command is for the Owner only.
This simply shows the channels that the bot attempts to join when it has connected to dAmn.

!autojoin add [channel]

This command is for the Owner only.
Use this command to add a channel to the bot's list of autojoin channels, where [channel] is the name of the channel you wish to add to the list. For example, type !autojoin add #Botdom to add #Botdom to the list of autojoin channels.

If [channel] is not provided, then the channel you used the command in will be used.

!autojoin remove [channel]

This command is for the Owner only.
Use this command to remove a channel from the bot's list of autojoin channels, where [channel] is the name of the channel you wish to remove from the list. For example, type !autojoin remove #Botdom to remove #Botdom from the list of autojoin channels.

If [channel] is not provided, then the channel you used the command in will be used.

!user list

This command is for Operators and up only.
This command shows a list of the users in the bot's user groups.

!user add [user] [group]

This command is for Operators and up only.
Use this command to add a user to a user group in the bot, where [user] is the name of the user you wish to add, and [group] is the name of the user group you want to add them to. For example, to add photofroggy to the user group Operators, type !user add photofroggy Operators.

!user remove [user]

This command is for Operators and up only.
Use this command to remove a user from the bot's user groups, where [user] is the name of the user you wish to remove, For example, to remove photofroggy, type !user remove photofroggy.

!users

This is the same as doing !user list, but this command is open to Guests and above.

!group list

This command is for the Owner only.
This command shows a list of the bot's user groups

!group add [group]

This command is for the Owner only.
Use this command to add a user group to the bot's user group system, where [group] is the desired name for the user group. For example, to add user group Biscuits, type !group add Biscuits.

!group delete [group]

This command is for the Owner only.
Use this command to delete a user group in the bot's user group system, where [group] is the desired name for the user group. For example, to add user group Biscuits, type !group add Biscuits.

!group alias add [group] [alias]

This command is for the Owner only.
You can use this command to add an alias for a user group, where [group] is the name of the user group you want to add an alias to, and [alias] is the alias you want to add to the user group. For example, to give the user group "Biscuits" the alias "Digestive", type !group alias add Biscuits Digestive.

!group alias remove [group]

This command is for the Owner only.
You can use this command to remove a user group's alias, where [group] is the name of the user group you want to remove the alias for, or the alias itself. For example, to remove user group "Biscuits"'s alias "Digestive", you can type either !group alias remove Biscuits or !group alias remove Digestive.

!group shift up [group]

This command is for the Owner only.
This command relates to the order in which the user groups are displayed when using the "!users" command. This command would move [group] up in the list by one place, where [group] is the name of the user group you want to move.

!group shift down [group]

This command is for the Owner only.
This command is the opposite of !group shift up. This command would move [group] down in the list by one place, where [group] is the name of the user group you want to move.

!group defaults

This command is for the Owner only.
This command resets all user group data! Use at your own risk!

!groups

This is the same as using !group list, but this command is open to Guests and above.

!eval [expression]

This command is for the Owner only.
This command evaluates a Python expression, and returns the result, where [expression] is a valid Python expression.

!exec [code]

This command is for the Owner only.
This command is similar to !eval, but it can execute valid Python codes as opposed to evaluating an expression , where [code] is valid Python code.

!sudo [user] [command] [command arguments]

This command is for the Owner only.
This command lets you execute a bot command on someone else's behalf, where [command] is the name of the command to execute, and [command arguments] are any arguments you want to pass to the command. As an example, to execute the !build command as "photofroggy", type !sudo photofroggy about.

!quit

This command is for the Owner only.
This command shuts down the bot and closes the window it is running in, without warning!

!quit soft

This command is for the Owner only.
This command shuts down the bot, but gives the owner the option of restarting the bot from the command line/terminal.

!restart

This command is for the Owner only.
This command restarts the bot!
