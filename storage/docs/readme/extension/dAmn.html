Extension: dAmn

Included in: All packages
Default status: On

The dAmn extension is an important extension, but is not required. The extension provides a set of commands that let you make your bot interact with dAmn, and also lets you view information about the bot's connection to dAmn. It would probably be best to keep this one loaded in your bot, as you may find it useful at some point, and it's always helpful to play around with it, just to see how certain things work.

This extension is contained in the "dAmn.py" file, located in the folder ~/extensions, meaning it is loaded into the bot on startup.

!say [channel] [message]

Operators and up
Use this command to make the bot say something, where [channel] is the channel to say the message in, and [message] is the message to say.

[channel] must start with either an @ or a # to be recognized as a channel name. If it does not start with an @ or a #, then it will be included as part of the message. If [channel] is not provided, then the message will be sent to the room you used to command in.

As an example !say #Botdom Hello there! would make the bot say "Hello there!" in the channel "#Botdom", whereas !say Botdom Hello there! would make the bot say "Botdom Hello there!" in the channel you used the command in.
Note: The bot must be in the channel defined by [channel] for this command to work properly.

Also keep in mind that using an @ signifies a private chat when using Terra.

The above applies to all commands where [channel] appears as the first parameter, if it is optional, of course.

!action [channel] [action]

Operators and up
Use this command to make your bot perform an action in a channel. This is pretty much the same as !say, but this sends an action rather than a normal message. It's like making the bot use /me on dAmn.

!me [channel] [action]

Operators and up
This is an alias for !action.

!join [channel]

Operators and up
Use this command to make your bot join a channel, where [channel] is the name of the channel you want your bot to join.

!part [channel]

Operators and up
This command makes the bot leave a room, where [channel] is the name of the channel you want the bot to leave. If [channel] is not provided, then the bot will leave the channel the command was used in.

!kick [channel] [user] [reason]

Operators and up
Kick a user from a channel, where [user] is the name of the user you want to kick. If [reason] is provided, the bot will attempt to kick the user with [reason] as the reason. For example, !kick photofroggy would kick photofroggy without a reason, while !kick photofroggy stop spamming would kick photofroggy with "stop spamming" as the reason.

!promote [channel] [user] [privilege class]

Operators and up
Promote a user in a channel, where [user] is the name of the user to promote, and [privilege class] is the name of the privilege class to promote them to. If [privilege class] is not provided, the user will be promoted by one privclass.

!demote [channel] [user] [privilege class]

Operators and up
This is the opposite of !promote.

!ban [channel] [user]

Operators and up
Use this command to ban a user from a channel, where [user] is the name of the user you want to ban.

!unban [channel] [user]

Operators and up
This is the opposite of !ban

!get [channel] [property]

Operators and up
Use this command to retrieve a channel's property. Argument [property] has to be either "title", "topic", "privclasses" or "members", or the bot will receive an error from dAmn. The bot will not display a message on dAmn after using this command, but you can check the console to see whether it was successful or not.

!set [channel] [property] [value]

Operators and up
Use this command to set a channel's property. Argument [property] has to be either "title" or "topic", or the bot will receive an error from dAmn. Argument [value] should be the value you want to give the property. For example, !set topic Hello, world! would set the topic to "Hello, world!".

!title [channel]

Operators and up
Use this command to view the title in [channel]!

!title [channel] set [title]

Operators and up
Use this command to set the title in [channel] to [title]!

!title [channel] append [text]

Operators and up
Use this command to append [text] to the title in [channel]!

!title [channel] replace [search] [replace]

Operators and up
Use this command to replace [search] with [replace] in the title in [channel]!

!title [channel] clear

Operators and up
Use this command to clear the title in [channel]!

!topic [channel]

Operators and up
Use this command to view the topic in [channel]!

!topic [channel] set [topic]

Operators and up
Use this command to set the topic in [channel] to [topic]!

!topic [channel] append [text]

Operators and up
Use this command to append [text] to the topic in [channel]!

!topic [channel] replace [search] [replace]

Operators and up
Use this command to replace [search] with [replace] in the topic in [channel]!

!topic [channel] clear

Operators and up
Use this command to clear the topic in [channel]!

!admin [channel] [command]

Operators and up
Perform an admin command in [channel], where [command] is the command you want to perform. Be careful with this one. It is the same as doing /admin from the web client.

!kill [user] [reason]

Owner
Use this command to kill a user's connection, where [user] is the name of the user whose connection you want to kill. Providing [reason] is similar to providing a reason when kicking a user from a channel. This command will only work properly if your bot has privileges to kill connections. In most cases, your bot does not have this privilege.

!raw [packet]

Operators and up
This command can be used to send a raw packet to dAmn! Only use this command if you know what you are doing.

!refresh [channel]

Operators and up
You can use this to make the bot disconnect from dAmn, and then reconnect. The bot will display a message on dAmn before refreshing its dAmn connection. If [channel] is provided, the bot will attempt to display this message in the channel defined.

!ping

Operators and up
Use this command to test the speed of the bot's connection to dAmn.

!channels [channel]

Members and up
You can use this command to view a list of the channels that the bot is currently in. As with !refresh, if [channel] is provided, the bot will attempt to display the list in the channel defined.

!connected [channel]

Use this command to see how long the bot has been connected to dAmn for.

!disconnects [channel]

Use this command to see how many times the bot has been disconnected while running.

!viper [channel]

View brief information about the build of dAmn Viper being used by the bot.

!whois [channel] [user]

Get whois information on a user, where [user] is the name of the user you want to get information on.

!members [channel]

View a list of the users in [channel]. If [channel] is not provided, then a list of the users in the channel you used the command in will be shown.
